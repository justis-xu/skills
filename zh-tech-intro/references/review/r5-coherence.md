# R5：整体连贯性（第一步跑）

**每次审查的第一步**：先整体读一遍，再进行逐条检查。结构性断层要先发现——后面的 R1~R4 是在内容连贯的基础上做的。

## 连贯性检查要点

**章节之间**
- 相邻章节是否有逻辑关联？结束一节时读者是否知道"接下来为什么要讲下一节"？
- 有没有章节突然切换话题，让读者感到"这怎么跳到这里了"？

**段落之间**
- 段与段之间有没有无铺垫的概念切换？
- 引入新概念前，前一段有没有为它铺垫？

**前后一致性**
- 某个机制/数字在文章多处提及，说法是否一致？
- 说了"有 N 种情况"的地方，是否实际讲了 N 种？

**读者视角**
- 如果读者只读过前面章节、没有源码背景，能看懂每节吗？
- 有没有某个词/概念突然出现，前面没有任何铺垫？
- 某节是否依赖"读者知道 X"才能理解，而 X 在后面才解释？

## 常见断层信号

| 信号 | 处理方式 |
|------|---------|
| 读者读完这节，还不知道"接下来讲什么" | 加过渡句 |
| 某个词/概念突然出现，前面没有铺垫 | 往前加一句引入 |
| 前面说了"有三种情况"，后面只讲了两种 | 补齐或删掉那句"三种" |
| 某节的开头和上一节的结尾完全没有关联 | 加一句衔接 |
| 同一个机制在不同章节的描述有矛盾 | 统一说法 |

## 报告格式

- 发现断层 → 说明位置（章节名/段落位置）+ 问题描述 + 候选修法
- 未发现问题 → 一句话确认"连贯性 clean，无断层"
