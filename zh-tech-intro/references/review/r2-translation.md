# R2：翻译腔审查

参考：`../language/baoyu-translate.md` refined 模式 Step 4（critique）的诊断标准

## 执行步骤

逐段读文章，重点查四类欧化语言：

| 欧化类型 | 典型信号 | 判断方法 |
|---------|---------|---------|
| **介词结构直译** | 通过...来...、对...进行...、基于...的 | 去掉介词短语，中文意思还完整吗？完整 → 介词多余 |
| **因果结构倒置** | "X，是因为 Y" | 中文习惯先说原因：把"是因为"改成句号，Y 放前面 |
| **名词化动词** | "进行压缩"、"实现了...的功能" | 直接用动词："压缩"；删掉"实现了...的" |
| **被动语态过度** | "被...所决定"、"由...来完成" | 改主动："X 决定"、"X 完成" |

## 额外检查项

- **直译英文技术词**：如"记忆预取"（prefetch 直译，硬件术语感）→ 改"后台记忆检索"
- **直译英文结构**：如"两条异步旁支"（async side branch 直译）→ 改"两个后台任务"
- **文学化词汇错位**：如"后面才不会迷失"（技术场景里太文学）→ 改"后面才不会绕"
- **英文动词名词化**：如"高效清桌"→ 改"快速完成上下文压缩"
- **送给模型**描述 API 调用：如"送给模型看的历史消息"→ 改"模型调用的输入消息"
- **引号强调已有描述**（冗余引号）：如`准备好"这一轮的历史消息"`→ 去掉引号，直接说清楚

## 报告格式

问题句全文 + 行号 + 2-3 个候选改法，候选之间说明语感差异。
