# 项目问卷生成指南

Step 1 的核心动作：**先快速扫系统，再生成针对这个系统的具体问题**。

不要用抽象的「读者类型 R1/R2/R3」替代具体问题——用户看不懂这些标签。用具体的、带选项的问题代替。

---

## 第一步：快速扫系统（5 分钟）

浏览源码目录、README，找到：
- 系统的 3~5 个核心概念（每一个都可能成为问题）
- 用户可能已经熟悉的周边技术（成为读者背景问题）
- 作者可能不清楚的实现细节（成为作者知识问题）

---

## 第二步：生成问卷

生成 **10~15 个问题**，覆盖以下五个维度，每个问题必须有 **3~4 个可选项**。

### A. 读者对核心概念的熟悉程度（4~6 题）

针对这个系统最重要的 4~6 个概念各出一题。

格式：
```
N. 读者是否了解「[系统专属概念]」？
   A) 没听说过
   B) 知道大概是什么
   C) 用过/接触过
   D) 深入了解/开发过类似系统
```

例（openclaw 记忆系统）：
```
1. 读者是否了解 AI Agent 的基本执行方式（感知→决策→行动循环）？
   A) 完全不了解  B) 知道大概  C) 用过相关工具  D) 开发过 Agent 系统

2. 读者知道上下文窗口（Context Window）是什么、会被撑满吗？
   A) 不知道  B) 听说过但不清楚细节  C) 了解限制和影响

3. 读者对向量检索（embedding 搜索）了解多少？
   A) 不了解  B) 知道概念  C) 实际用过  D) 知道实现原理

4. 读者有没有用过 Cursor、Claude Code 等 AI 编程工具？
   A) 没有  B) 偶尔用  C) 每天用，熟悉它的记忆/上下文行为
```

### B. 写作目的（2~3 题）

```
N. 这篇文章的主要目标是什么？（可多选）
   A) 让读者知道这个系统存在、解决什么问题
   B) 吸引开发者使用或参与贡献
   C) 展示核心设计的精妙，供技术读者深度参考
   D) 帮助已有用户更好地使用、配置这个系统

N. 文章的深度定位？
   A) 知道有什么功能就够了，不需要懂实现
   B) 要能理解设计思路和关键取舍
   C) 要能看懂参数来源、公式和实现细节
```

### C. 作者对系统各模块的了解程度（3~5 题）

针对系统的主要模块各出一题，让作者告诉你哪些模块他清楚、哪些需要你来挖。

格式：
```
N. 你对「[模块名]」的了解程度？
   A) 需要你来研究，我不清楚
   B) 大致了解，但细节不确定
   C) 比较清楚，可以直接告诉你
   D) 读过源码，非常清楚
```

例（openclaw 记忆系统）：
```
5. 你对三个文件（日记/DREAMS.md/MEMORY.md）的分工熟悉吗？
   A) 需要你来研究  B) 大致了解  C) 比较清楚  D) 读过源码

6. 你对梦境三阶段的调度机制（什么时候触发、怎么接力）了解吗？
   A) 不清楚  B) 大致知道  C) 清楚，包括统一调度的架构变化

7. 你对六维升级评分公式的了解？
   A) 不知道有这个  B) 知道有评分，不清楚维度  C) 知道维度和权重  D) 读过源码，知道每个公式
```

### D. 文章语言（1 题）

```
N. 文章语言？（根据用户输入语言推荐，用户确认）
   A) 中文（推荐，基于你的输入）
   B) 英文
   C) 其他：___
```

### E. 文章模式（1 题，结合 modes.md）

```
N. 文章侧重哪个方向？
   A) 快速入门——是什么、怎么用，跳过实现细节（Mode A）
   B) 架构深挖——核心设计决策 + 关键参数，含源码（Mode B）
   C) 方案对比——和同类方案横向比较，帮助选型（Mode C）
```

---

## 第三步：写入 context.md

用户回答后，把结果写入文章所在目录的 `context.md`。

### context.md 格式

```markdown
# 写作上下文

## 系统
[系统名称和一句话定位]

## 文章语言
[中文 / 英文 / 其他]

## 文章模式
[A / B / C] — [一句话说明]

## 写作目标
[根据用户选择整理成一句话]

## 读者画像
| 概念 | 读者熟悉程度 |
|------|-------------|
| [概念1] | [不了解 / 知道概念 / 用过 / 深度了解] |
| [概念2] | ... |

## 作者知识储备
| 模块 | 了解程度 | 备注 |
|------|---------|------|
| [模块1] | [需研究 / 大致了解 / 比较清楚 / 读过源码] | |
| [模块2] | ... | |

## 特别说明
[用户补充的任何其他要求]
```

---

## 第四步：后续会话加载

再次打开同一篇文章时，先检查文章目录是否有 `context.md`：
- 有 → 加载，告知用户「已加载上次的写作上下文，如需修改说一声」，直接进入流程
- 没有 → 走完整问卷流程
