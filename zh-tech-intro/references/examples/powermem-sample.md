## 前言（完整示例）

> ⚠️ 注意：这是正确的前言写法。SCQA 框架是内化的，不出现"核心矛盾"/"解法要点"/"深度预告"等标签作为标题。文章没有大标题，第一章直接是 `# 一、前言`。

PowerMem 是一个为 AI Agent 设计的**持久化智能记忆层**。核心价值是：用 Ebbinghaus 遗忘曲线管理记忆生命周期，用混合搜索实现高质量召回，让 AI 真正具备"跨会话长期记忆"的能力。

一句话概括：**它不是向量数据库，而是让记忆像人类一样会遗忘、会强化、会压缩的 AI 第二大脑。**

LOCOMO 长对话记忆基准测试验证了这套设计的有效性：

| 维度          | PowerMem   | 全量上下文 | 提升幅度   |
| ------------- | ---------- | ---------- | ---------- |
| 准确率        | **78.70%** | 52.9%      | +48.8%     |
| 检索 p95 延迟 | **1.44s**  | 17.12s     | 11.9× 更快 |
| Token 用量    | **~0.9k**  | ~26k       | 28.9× 更少 |

PowerMem 围绕两条主干流程构建：`add()` 写入记忆 和 `search()` 检索记忆。

* **写入侧**的核心问题是：**原始对话不能直接存**。用户输入充满噪声、重复、矛盾，直接存向量库只会让记忆库越堆越乱。PowerMem 的解法是用 LLM 先提炼出结构化事实，再通过四类决策事件（ADD/UPDATE/DELETE/NONE）精确地管理每条记忆的生死。

* **检索侧**的核心问题是：**单路向量搜索召回不全**。语义相似 ≠ 关键词匹配，专有名词尤其容易在向量空间被稀释。PowerMem 的解法是三路混合搜索（dense 向量 + 全文 + sparse 向量），叠加 Reranker 精排，再用 Ebbinghaus 遗忘分数做最终加权。

这两条流程之外，还有两个贯穿全局的**后台机制**：

- **遗忘曲线**：记忆不是永久保存的。每次 `search` / `get` 都会触发生命周期判断，衰减过快的记忆会被自动清除，频繁命中的记忆会被晋升强化。
- **记忆压缩**：定期去重和聚类压缩，防止记忆库无限膨胀，保持搜索质量。

## 写入流程（配图示例）

PowerMem 的 `add()` 是一个 **LLM 驱动的语义提炼流程**，而不是简单的"写入向量库"。下图是完整的写入链路：

```
# 画图提示：横向阶段列流程图，展示 PowerMem 写入记忆的完整链路。
#

#   ① 「预处理」
#   ② 「事实提取」
#   ③ 「事件决策」
#   ④ 「重要性评分」
#   ⑤ 「写入存储」
#
# 各列纵向节点：

#   第②列：「提取结构化事实」→「拆分为自包含语义单元」
#   第③列：「检索相关旧记忆」→「LLM 决策：ADD / UPDATE / DELETE / NONE」
#   第④列：「6 维加权评估」→「输出重要性分数」→「生成艾宾浩斯元数据」
#   第⑤列：「向量化事实文本」→「落库（含元数据）」
#
# 列间连接：
#   第①列底部「是」出口 → 第②列顶部「提取结构化事实」
#   第②列底部 → 第③列顶部

#
# 风格：白色背景，高级技术 PPT 配图，极简专业，文字标注全部用中文。
```

**流程可以分为三个阶段来理解：**

1. **预处理**：去重、重要性初步评估，判断是否走 LLM 流程
2. **LLM 推理**：事实提取、相似记忆检索、四类事件决策
3. **写入存储**：重要性计分、Ebbinghaus 元数据生成、向量化落库

每一条通过事实提取产生的"记忆单元"，都会经历完整的三阶段，最终只有真正有价值的信息才会写入向量库。

## 步骤：事实提取（加粗疑问句 + 代码块示例）

> ⚠️ 注意：这里展示"为什么"用加粗疑问句引出，而不是开独立小节（`###`）。

**为什么不直接存原始消息？**

对话消息通常是这样的：

> "嗯对，我之前说的那个，就是我不太喜欢辣，昨天去吃串串结果辣到不行，对了你帮我记一下我老家在成都。"

这条消息里混杂了情绪表达、事件描述、重要事实。如果直接存进向量库，检索"用户口味偏好"时会把整段话召回，噪声极大。

**事实提取的目标**就是从这类杂乱输入中蒸馏出结构化的、自包含的语义单元：

```json
{
  "facts": [
    "用户不喜欢辣食",
    "用户老家在成都",
    "用户昨天吃了串串"
  ]
}
```

PowerMem 使用 `FACT_RETRIEVAL_PROMPT` 驱动 LLM 完成这个任务。Prompt 的核心规范包括：

- **完整性**：每条事实必须是自包含的，包含时间、地点等上下文（"2023年5月去了夏威夷"，而不是"去了夏威夷"）
- **时序保留**：相对时间引用（"昨天"、"上周"）必须原样保留，不能丢失
- **意图捕获**：用户的计划和意图也要提取（"想预约心内科"），不仅限于已发生的事实
- **语言不翻译**：输入是中文输出就是中文，不做语言转换

## 关键设计：遗忘曲线（公式 + 分级卡片配图 + 引用块总结示例）

> ⚠️ 注意：这里展示哲理性结论放 `>` 引用块，而不是嵌在段落末尾。

**为什么需要遗忘？**

一个没有遗忘机制的记忆系统会无限堆积记忆。运行一年后，记忆库里可能有几十万条条目——其中大量是过期信息（"今天天气很好"已经是一年前的事）和低价值闲聊碎片。如果一并召回，模型很难判断哪条信息更可信、更新、更相关。

但如果简单地删除旧记忆，又会丢失用户偏好变化的历史轨迹。PowerMem 的思路是让记忆像人类大脑一样自然衰减：

> 重要的记忆被保留和强化，低价值记忆逐渐淡出检索结果。

根据 Ebbinghaus 提出的遗忘曲线，核心数学公式如下：

$$R = e^{-\frac{t}{S}}$$

| 变量 | 名称 | 说明 |
| :--: | :--- | :--- |
| $R$ | 记忆保留率 | 取值 $[0.0, 1.0]$，当前记忆留存比例 |
| $t$ | 经过时间 | 自记忆创建起所经过的小时数 |
| $S$ | 记忆强度 | $S = 24 / \text{decay\_rate}$，值越大遗忘越慢 |

```
# 画图提示：左右分栏布局，左侧展示遗忘曲线公式与折线图，右侧展示三种记忆类型分级卡片。
#
# 左侧（约 1/2 宽）：
#   大号公式框：「R = e^(−t/S)」
#   变量说明表：R = 记忆保留率 / t = 经过时间（小时）/ S = 记忆强度
#   折线图：X轴「经过时间（天）」，Y轴「记忆保留率（0~1.0）」
#     三条曲线，从陡到缓依次标注：「工作记忆」「短期记忆」「长期记忆」
#
# 右侧（三张卡片并排，各对应一种记忆类型）：
#   卡片1：「工作记忆」
#     条件：重要性分数 < 0.3
#     关键参数：衰减速率最快
#   卡片2：「短期记忆」
#     条件：0.3 ≤ 重要性分数 < 0.8
#     关键参数：衰减速率中等
#   卡片3：「长期记忆」
#     条件：重要性分数 ≥ 0.8
#     关键参数：衰减速率最慢
#   三张卡片颜色与左侧折线一一对应
#
# 风格：白色背景，高级技术 PPT 配图，极简专业，文字标注全部用中文。
```

不同记忆类型使用不同的衰减速率：

| 记忆类型 | 衰减速率 | 记忆强度 $S$ | 遗忘速度 |
|---|:---|:---|---|
| 工作记忆 | 0.20 | 120 小时 | 较快 |
| 短期记忆 | 0.15 | 160 小时 | 中等 |
| 长期记忆 | 0.10 | 240 小时 | 较慢 |

例如 $S = 120$ 小时，表示约 120 小时后，记忆会衰减到初始强度的 36.8%；约 3.5 天后衰减到一半。

> 你不会记得三天前的每句寒暄，但你会记得自己的生日和家人的名字。记忆不是被粗暴删除，而是像人类大脑一样，随着时间自然遗忘，并在需要时被重新强化。
