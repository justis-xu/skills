# 技术术语翻译参考

> **使用时机**：正文出现英文词时，先查豁免列表（SKILL.md 规则 20）；不在豁免列表的，查此文件找同类处理方法，再决定翻译还是首次标注。

---

## 一、翻译核心原则：看行为，不看词面

直译英文技术词往往产生"机器翻译腔"。正确做法是先弄清楚这个词在当前语境里描述的是什么行为，再用中文的行为描述替代。

**判断流程：**
1. 删掉这个英文词，用一句话说清楚它在描述什么
2. 那句话里核心的动词或名词就是中文的来源
3. 验证：把翻译结果给没看过文章的人，他能猜到原意吗？能 → 好翻译；不能 → 还是直译味

**拿不准时**：列出 2～3 个候选说法，加一句描述语感差异，让用户选，不要猜一个凑合用。

## 二、按类型分类处理

### 2.1 执行/操作类名词

| 英文词 | 直译（避免） | 按行为（推荐） | 说明 |
|--------|------------|--------------|------|
| attempt | 尝试 | 一次执行 / 执行尝试 | 是"执行单元"时→"一次执行"；是"重试"时→"再次发起" |
| compact | 压缩 | 压缩（保留，行为直观） | 动词/名词均直接用"压缩" |
| failover | 故障转移 | 故障转移（保留，约定俗成） | 已有中文，直接用 |
| rollback | 回滚 | 回滚（保留，技术社区通用） | 直接用，无需另译 |

### 2.2 状态/模式类形容词

| 英文词 | 直译（避免） | 按行为（推荐） | 说明 |
|--------|------------|--------------|------|
| idle | 空闲 | 空响应（AI 语境）/ 静默 | "idle timeout"在 AI 语境里是"模型长时间无输出"，译为"空响应超时"比"空闲超时"准确 |
| stale | 过期 | 失效 | "stale cache"→"失效的缓存"；"失效"强调功能丢失，"过期"暗示时间到了 |
| blocking | 阻塞 | 阻塞（保留）/ 同步等待 | 技术读者用"阻塞"，非技术读者用"同步等待" |

### 2.3 数据结构/单元类名词

| 英文词 | 直译（避免） | 按行为（推荐） | 说明 |
|--------|------------|--------------|------|
| span | 跨度 | 消息段 / 追踪片段 | 可观测性语境→追踪片段；消息处理语境→消息段（一组相关消息的整体） |
| transcript | 记录 | 对话记录 / 消息序列 | 通常指实时消息历史；与"历史文件"区分时用"对话记录" |
| snapshot | 快照 | 快照（保留，约定俗成） | 直接用 |
| chunk | 块 | 分块 / 片段 | "chunking"→"分块处理"；"a chunk of text"→"一段文本" |
| batch | 批次 | 批次（名词）/ 批量（形容词/动词） | — |

### 2.4 配置/凭证类名词

| 英文词 | 直译（避免） | 按行为（推荐） | 说明 |
|--------|------------|--------------|------|
| profile | 配置文件 | 认证配置 / 用户信息（视语境） | auth profile→认证配置；user profile→用户信息 |
| credential | 凭证 | 凭据 / 密钥信息 | "凭据"比"凭证"更口语 |

### 2.5 系统组件/通道类名词

| 英文词 | 直译（避免） | 按行为（推荐） | 说明 |
|--------|------------|--------------|------|
| lane | 车道 | 通道 | "session lane"→会话通道；"global lane"→全局通道 |
| pipeline | 管道 | 管道（保留）/ 处理流程 | 架构描述用"管道"；强调步骤时用"处理流程" |
| engine | 引擎 | 引擎（保留，已通用） | "context engine"→上下文引擎 |
| provider | 提供者 | 提供商（第三方服务）/ 提供者（依赖注入） | 区分语境 |

### 2.6 生命周期/回调类名词

| 英文词 | 直译（避免） | 按行为（推荐） | 说明 |
|--------|------------|--------------|------|
| afterTurn / post-turn | 轮后处理 | 轮末处理 | "after turn ends"→轮末；"轮后"读起来别扭，用"轮末" |
| maintain | 维护 | 维护（保留，行为直观） | "维护历史记录"直接用"维护" |

## 三、AI / LLM 领域速查

| 英文词 | 处理方式 | 说明 |
|--------|---------|------|
| prompt | **豁免，直接用** | 中文技术社区直接说 prompt，"提示词"显得生硬 |
| System Prompt | **豁免，直接用** | 大写作专有概念名，与 prompt 同理 |
| embedding | **豁免，直接用** | 中文 AI 社区直接说 embedding |
| thinking block | 思考块 | 模型产生的内部思维内容块；"推理过程块"偏逻辑推断语气，"思考块"更贴近 thinking 的含义 |
| context window | 上下文窗口 | 已有约定俗成中文，直接用 |
| fine-tuning | 微调 | 约定俗成，直接用 |
| retrieval | 检索 | 直接用"检索"，不用"取回" |
| turn | 轮次 / 轮 | "one turn"→一轮；"user turn"→用户轮 |
| session | 会话 | 直接用"会话" |
| DM | 私信 | Direct Message；用"私信"，不用"直接消息" |
| channel | 频道 | 群组/频道场景用"频道" |
