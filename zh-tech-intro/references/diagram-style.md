# 画图风格参考指南

> 生图工具：**gpt-image-2**（经实践验证）

---

## 一、核心原则

画图提示是给 AI 生图工具的 **prompt**，核心是描述「展示什么信息、用什么结构呈现」。

### 绝对禁止写入 prompt 的内容

| 禁止 | 原因 |
| --- | --- |
| CSS 颜色值（`#3B82F6`、`rgba(59,130,246,0.5)`） | 过度限制模型发挥，生成结果反而变差 |
| 像素/尺寸（`800px`、`1200×600`、`14pt`、`2px`） | 生图模型不接受绝对尺寸，写了等于没写 |
| 字号/字重（`bold`、`font-size: 16`） | 同上，渲染细节由模型决定 |
| SVG/HTML/CSS 代码片段 | 不是代码生成任务 |
| 箭头粗细、边框样式等渲染细节 | 描述内容和结构，不描述样式参数 |

### 应该写的内容

- **图类型**：横向多层架构图、竖向三阶段流程图、条形图+卡片……
- **节点标签**：全部用中文，描述语义（「同步决策」而非 `SyncDecision`）
- **箭头语义**：说明关系（「降级路由」「触发迁移」），不说方向
- **关键数据**：阈值、比例、数字直接写进来（「得分 0.62」「权重 30%」）
- **视觉层次**：「重点突出」「虚线框表示可选」「深色背景强调」

---

## 二、【禁用】泳道图

**不要使用纵向泳道图（Vertical Swimlane）。**

gpt-image-1 实践验证：纵向泳道图 prompt 生成结果极差——泳道线混乱、节点溢出边界、文字截断、整体像线框图而非正式图表。

**替代方案**：

| 原泳道图场景 | 替换为 |
| --- | --- |
| 跨模块的完整流程（跨角色） | 竖向三阶段流程图（§3.2） |
| 数据在多层/多组件间流转 | 横向多层架构图（§3.1）或横向阶段列流程图（§3.7） |
| 强调「谁做什么」的分工 | 横向阶段列流程图（§3.7），顶部加阶段标签 |

## 三、图类型参考模板

### 3.1 横向多层架构图

适用：展示系统分层结构、各层组件和整体数据流。  
参考图：`assets/01-architecture.png`（MemoryCore 四层架构）

```
# 画图提示：横向多层架构总览图，展示[系统名]的[N]层结构与数据流。
#
# 内容描述（从上到下）：
#   第一层【[层名]】（浅色背景）：
#     并排输入块：「[节点A]」「[节点B]」
#   第二层【[层名]】（浅蓝背景）：
#     节点串联：「[步骤1]」→「[步骤2]」→「[决策节点]」
#     决策节点分叉为三路：「[分支A]」「[分支B]」「[分支C]」
#   第三层【[层名]】（浅绿背景）：
#     两个区域并排：
#       主区域（圆角矩形）：「[名称]」内含「[子组件1]」和「[子组件2]」
#       备用区域（虚线框）：「[名称]」
#     两者之间虚线箭头标注「[语义]」
#   第四层【[层名]】（浅橙背景）：
#     节点串联：「[步骤1]」→「[步骤2（可选）]」→「[步骤3]」
#   右侧独立模块（浅紫背景，竖排）：「[子系统名]」含三个子节点
#
# 风格：白色背景，高级技术 PPT 配图，极简专业，文字标注全部用中文。
```

### 3.2 竖向三阶段串行流程图

适用：有清晰阶段划分的处理流程，替代泳道图。  
参考图：`assets/02-dreaming-flow.png`（梦境三阶段）

```
# 画图提示：竖向三阶段流程图，展示[流程名]
#
# 顶部触发器：「[触发条件]」→ 判断菱形「[已完成检查]？」→ 是则结束，否则继续
#
# 三个串联阶段框（圆角矩形，从上到下）：
#   阶段一【[名称]】（浅蓝背景）：
#     深色标题栏：「[名称] · [副标题]」
#     内容：「[步骤1]」→「[步骤2]」→「[步骤3]」
#     输出：文档图标「[输出产物名]」
#   阶段二【[名称]】（浅紫背景）：
#     深色标题栏：「[名称] · [副标题]」
#     内容：「[步骤1]」→「[步骤2（LLM调用，虚线箭头）]」→「[步骤3]」
#     输出：文档图标「[输出产物名]」
#   阶段三【[名称]】（浅绿背景）：
#     深色标题栏：「[名称] · [副标题]」
#     内容：「[步骤1]」→「[步骤2]」→「[步骤3]」
#     输出：文档图标「[输出产物名]」
# 阶段间粗箭头向下，右侧有整体回流箭头标注「[全局触发语义]」→「[最终存储节点]」
#
# 风格：白色背景，高级技术 PPT 配图，极简专业，文字标注全部用中文。
```

### 3.3 横向条形图 + 说明卡片

适用：多维度权重或评分分布，每个维度附带机制说明。  
参考图：`assets/03-weights-bar.png`（6维权重总览）

```
# 画图提示：横向条形图 + 说明卡片布局，展示[N]个维度的权重分布
#
# 左侧：横向条形图，[N]行，每行对应一个维度
#   维度标签（左对齐）：「[维度1]」「[维度2]」…「[维度N]」
#   条形宽度对应权重（从大到小）：[w1] / [w2] / … / [wN]
#   条形颜色：深蓝（最高）渐变到浅灰（最低）
#   每条末尾标注百分比数字
#   左上方标题：「[图表标题]」副标题「[副标题]」
#
# 右侧：[N]张说明卡片，2列排列
#   每卡含：图标 + 维度名（加粗）+ 计算逻辑一句话 + 防作弊机制说明（盾牌图标）
#
# 风格：白色背景，高级技术 PPT 配图，极简专业，文字标注全部用中文。
```

### 3.4 左右分栏：公式 + 分级条件卡片

适用：展示公式的分级/分支行为，配折线图直观呈现。

```
# 画图提示：左右分栏布局，左侧展示公式与折线图，右侧展示分级条件卡片。
#
# 左侧：
#   大号公式框：[公式，用中文描述变量]
#   变量说明表：[变量名（中文）] 对应 [含义]
#   折线图（可选）：X轴「[含义]」，Y轴「[含义，范围]」
#   [N]条曲线，标注各自场景名
#
# 右侧：[N]张条件卡片并排
#   每卡：条件标题（「[参数] [≥/</=] [阈值]」）+ 场景名 + 关键参数值
#   卡片颜色与左侧折线一一对应
# 底部说明框：「[一句话总结]」
#
# 风格：白色背景，高级技术 PPT 配图，极简专业，文字标注全部用中文。
```

### 3.5 并排方案对比图

适用：两个方案/版本的流程对比，突出差异点。  
参考图：`assets/06-agent-comparison.png`（无Agent vs 有Agent）

```
# 画图提示：左右并排对比图，展示[方案A]与[方案B]的差异。
#
# 顶部各贴对比标签（深色背景标题框）：
#   左侧：「[方案A特征，如「无Agent·直连检索」]」
#   右侧：「[方案B特征，如「有Agent·智能路由」]」
# 各自按编号排列步骤：
#   左侧：① → ② → ③（步骤少，流程简单）
#   右侧：① → ② → 三路分支（各分支含关键数字）→ 汇合 → ③ → ④
# 底部横跨两侧对比框：「核心差异：[关键指标对比，如「准确率 +X%（A% vs B%）」]」
#
# 风格：白色背景，高级技术 PPT 配图，极简专业，文字标注全部用中文。
```

### 3.7 横向阶段列流程图

适用：多个处理阶段，每个阶段内有纵向子流程，阶段间横向推进。  
参考图：`assets/05-write-chain.png`（OpenViking 写入完整链路）

```
# 画图提示：横向阶段列流程图，展示[系统名]的完整[流程名]。
#
# 顶部：[N]个阶段标题列，编号 ①②③④⑤，每列一个深色背景标题：
#   ① 「[阶段1名]（[副标题]）」
#   ② 「[阶段2名]（[副标题]）」
#   … 依此类推
#
# 各列纵向节点（圆角矩形，从上到下）：
#   第①列：[节点A] → [节点B] → [节点C]
#   第②列：[节点D] → [节点E]（含判断分支）→ [节点F]
#   第③列：[节点G]（跨行大节点）
#   第④列：[节点H] → [节点I] → [节点J]
#   第⑤列：[节点K] → [节点L] → [节点M]（含异步分支，虚线）
#
# 箭头：
#   主流程（同步顺序）：实线→
#   LLM调用/异步触发：紫色或绿色虚线→
# 底部图例：说明各类箭头语义
#
# 风格：白色背景，高级技术 PPT 配图，极简专业，文字标注全部用中文。
```

## 五、节点标签规则

| 禁止 | 替换为 |
| --- | --- |
| 函数名（`add_memory()`） | 中文功能描述（「写入对话」） |
| 类名（`ShortTermMemory`） | 中文层名（「短期记忆」） |
| 字段名（`N_visit`） | 中文含义（「命中次数」） |
| 英文缩写（`FAISS`、`LFU`） | 中文描述（「向量索引」、「最少使用淘汰」） |
| 纯 `True / False` | 「连续 ✓」「不连续 ✗」 |
| `embedding`、`vector` | 「语义向量」 |
| CSS 颜色值（`#3B82F6`） | 描述性颜色名（「深蓝」「浅灰」） |
| 像素/尺寸值（`800px`、`14pt`） | **删除**，生图工具自行决定 |
